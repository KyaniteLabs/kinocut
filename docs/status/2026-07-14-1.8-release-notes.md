# Kinocut 1.8.0 release notes

**Published:** 2026-07-14 — [GitHub Release](https://github.com/KyaniteLabs/kinocut/releases/tag/v1.8.0). PyPI, npm, and the MCP Registry resolve to 1.8.0.

**Published surface:** 142 MCP tools / 121 CLI commands.

## Three agent-facing bullets

1. **What agents can do now** — Contract-first AI-video path on the public surface:
   ingest → preflight → temporal inspect → exact-asset verdict / acceptance →
   body-swap / salvage with lineage, plus optional C2PA on export and a staged MCPB
   package for Desktop install experiments.
2. **What got safer** — Approvals require active human decisions (analyzer-only cannot
   approve); salvage/body-swap reject stale/aliased/protected inputs; Hyperframes init
   no longer hangs under MCP (no TTY).
3. **Compat** — `mcp-video==1.6.2` installs current Kinocut 1.8.0; `mcp_video` imports,
   `MCP_VIDEO_*`, `~/.mcp-video`, `mcp-video://`, and legacy receipt keys remain supported
   through at least the 1.8.x line. Prefer `kinocut` / `kino` for new work.

Kinocut 1.8.0.

## Release-detail notes

### Added

- Governed AI-video foundation and review/salvage surfaces (MCP, CLI, Python).
- Optional C2PA signing for path-based exports (`c2patool`, off by default, verify-after-sign).
- Staged MCPB package + fail-closed native builder foundation (not a fully self-contained runtime).
- Path-based `kinocut-repurpose` skill + deterministic local demo.
- Field safety: loss-proof add-audio policies; authored ASS + dimension-aware subtitles.
- Hyperframes init fixed for non-interactive / MCP hosts.

### Sound program boundary

S13 host joins and S14 dual-class benchmark receipts may be complete in-tree; **S15
adversarial acceptance was STOP at release.** Full-episode Sonic World is not claimed
as shipped in 1.8.0.
